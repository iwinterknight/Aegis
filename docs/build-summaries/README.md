# `docs/build-summaries/` — Module Build Summaries

One write-up per completed module/phase, written **top-down at three altitudes**:

1. **Objective** — what the module is *for* in Aegis; which FRs and principles it serves.
2. **Functional** — how the pieces fit: data flow, components, the design decisions and tradeoffs.
3. **Syntax** — the salient lines of code: idioms, gotchas, why-this-not-that.

Each summary also:
- **Cites the concepts** from the deep dive (links to `../primers/`).
- **Calls out the SWE/MLOps principles** in play.
- Ends with **"Salient Points from Our Discussion"**, distilled from `../discussion-notes/`.

Naming: `phase-<N>-<slug>.md`.
