# `docs/primers/` — Concept Primers

One file per concept/technology, written top-down to enterprise depth:

1. **The problem it solves** — what breaks without it.
2. **Where it sits in Aegis** — which module/FRs/principles it serves.
3. **How it works under the hood** — the real mechanism, not a hand-wave.
4. **Industry/enterprise-standard usage** — how you'd apply it in production.

Naming: `<concept-slug>.md` — e.g. `paged-attention.md`, `radix-attention.md`, `raptor.md`,
`mcp-json-rpc.md`. Primers are concept-scoped (cross-phase), unlike build summaries which are
module-scoped. Link related primers with relative links.
