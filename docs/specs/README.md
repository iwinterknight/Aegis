# `docs/specs/` — System Specification

- **`Project_Aegis_System_Specification.md`** — the **living source of truth**. All design changes
  are made here as reviewable git diffs. Requirements are addressable IDs (`FR-IN-1`, `SG-1`, `P2`)
  that code, tasks, tests, and summaries trace back to.
- **`Project_Aegis_System_Specification_v4.3.docx`** — the original Word seed (Revision 4.3) the
  Markdown was converted from. Kept immutable for reference; do not edit — change the Markdown.

When the spec is revised, add a dated note to the top of the Markdown and (if the decision is
significant) a matching record in [`../adr/`](../adr/).
